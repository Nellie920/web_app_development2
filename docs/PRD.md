# PRD - 食譜收藏夾

## 1. 專案概述
- **背景與動機**：想要一個專屬的空間來記錄和收藏自己喜歡的食譜，並且希望畫面風格可愛圓潤，增加使用的愉悅感。同時也希望能夠跟其他人分享並獲得回饋。
- **目標用戶**：我自己（主要管理者/使用者），以及其他對食譜有興趣的訪客（可以按讚、評論、分享）。
- **核心價值主張**：提供一個美觀、易用且具備可愛風格的個人專屬食譜管理平台，並具備輕量級的社群互動功能。

## 2. 功能需求
- **主要功能**：
  1. 新增及刪除食譜
  2. 列出食譜的材料及步驟
  3. 提供可愛圓潤的使用者介面 (UI)
  4. 支援食譜圖片插入與展示
  5. 接受他人評論、按讚及分享

- **使用者故事**：
  - 作為**系統擁有者（我自己）**，我希望**能夠新增和刪除食譜**，以便**管理我的美食收藏**。
  - 作為**使用者**，我希望**能在食譜中清楚列出材料及步驟**，以便**做菜時能輕鬆參考**。
  - 作為**使用者**，我希望**能為食譜插入圖片**，以便**讓食譜紀錄更豐富吸引人**。
  - 作為**訪客**，我希望**能對喜歡的食譜按讚與發表評論**，以便**給予作者回饋與互動**。
  - 作為**使用者/訪客**，我希望**能分享食譜**，以便**推薦美味料理給親朋好友**。
  - 作為**所有使用者**，我希望**操作介面是可愛圓潤的風格**，以便**在使用時感到愉悅與放鬆**。

## 3. 非功能需求
- **技術限制**：使用 Flask (Python 後端) + Jinja2 (模板引擎) + SQLite (資料庫)。前端使用 HTML/CSS (Vanilla CSS 或搭配適當的輕量化框架實現可愛風格) 及 Vanilla JavaScript。
- **效能與安全考量**：
  - 需實作圖片上傳的大小與格式限制（如僅限 JPG/PNG），防止惡意檔案上傳。
  - 表單輸入需進行基本防護，避免 XSS 與 SQL Injection 攻擊。
  - 密碼（如未來需登入管理）需加密儲存。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 首頁列出所有食譜（包含縮圖）。
  - 新增/編輯/刪除食譜功能。
  - 單一食譜詳細頁（包含圖片、材料、步驟）。
  - 全站套用「可愛圓潤」風格的 CSS 設計。
- **Should Have (應該有)**：
  - 訪客按讚與評論功能。
  - 複製食譜網址的分享功能。
- **Nice to Have (可以有)**：
  - 依據食材或標籤搜尋食譜。
  - 一鍵分享至特定社群平台（如 Facebook, LINE）。
